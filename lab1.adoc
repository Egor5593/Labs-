== Цель

Цель отчета – узнать, как проходит процесс преобразования кода в исполняемый файл для микроконтроллера, как запускается программа и функция main() и т.д.

== Компиляция кода для нашего микроконтроллера

Компиляция кода на языке *C++* для микроконтроллера в *IAR Workbench for ARM* состоит из нескольких этапов.

=== Компиляция исходного кода в объектные файлы

На этом этапе *компилятор* преобразует исходный код в *объектные файлы*.

[NOTE]
====
Результаты компиляции:
* *Объектные файлы* (.o) с машинным кодом;
* *Библиотеки* (.lib), которые можно использовать в других проектах.
====

=== Линковка в исполняемый файл

Далее *линковщик* объединяет все объектные файлы, библиотеки и дополнительные данные в единый *исполняемый файл*.

[NOTE]
====
На этапе линковки используются:

* Объектные файлы;
* Внешние библиотеки;
* Стандартные библиотеки C++;
* Файл конфигурации памяти микроконтроллера.
====

Итог — создание исполняемого файла, готового к прошивке устройства.

=== Загрузка и отладка программы

После линковки формируется *ELF-файл*, который можно использовать для:
- Программирования микроконтроллера через отладчик или программатор.

=== Краткий итог процесса компиляции

[IMPORTANT]
====
Этапы компиляции:

* Компиляция исходного кода → объектные файлы;
* Линковка → исполняемый файл.
====

Далее исполняемый файл загружается в микроконтроллер для выполнения.

== Процесс запуска программы и функции main() на микроконтроллере

Перед вызовом main() система выполняет ряд подготовительных операций.

=== Подготовительные шаги перед main()

Инициализация стека и глобальных переменных → Инициализация периферийных устройств (таймеры, порты GPIO и др.) (выполняет программист) → Передача управления функции main().

=== Резюме запуска программы

[IMPORTANT]
====
До начала выполнения main() происходят:

* Инициализация памяти;
* Перенос данных из ПЗУ в ОЗУ;
* Настройка периферии;
* Переход к основной программе через main().   
====

== Что такое статическая переменная?

Статическая переменная в функции:

* Инициализируется только при первом вызове функции;
* Сохраняет своё значение между всеми последующими вызовами;
* Существует на протяжении всей работы программы.

== Работа статической переменной в функции

Пример кода: 

[source,cpp]
----
#include <iostream>

int Increment(int value)
{
    static int result = value;
    result++;
    return result;
}

int main()
{
    auto result = Increment(10);
    std::cout << "result == " << result << std::endl; // 11

    result = Increment(12);
    std::cout << "result == " << result << std::endl; // 12
}
----

=== Пояснение к примеру кода

При первом вызове Increment(10):

* result инициализируется значением 10.
* Значение увеличивается на 1 → result == 11.

При втором вызове Increment(12):

* result уже равно 11 (сохранилось с прошлого вызова).
* Значение снова увеличивается на 1 → result == 12.

[NOTE]
====
Параметр value влияет только на первую инициализацию статической переменной.
====

=== Ожидаемый вывод программы

[source,text]
----
result == 11
result == 12
----

== Почему код пишут самые низкоквалифицированные инженеры?

На мой взгляд, сегодня код зачастую пишут специалисты с невысокой квалификацией. Причина в том, что существует множество нейросетей, способных генерировать код на достойном уровне.

Основная задача разработчика — чётко понимать поставленную цель и уметь анализировать предложенные решения: 
* логичен ли код;
* требует ли он доработки или исправлений.

[IMPORTANT]
====
Таким образом, роль программиста всё чаще сводится не к написанию кода с нуля, а к корректировке, осмыслению и адаптации результатов, предложенных нейросетями.
====

== Краткий вывод

Я понял, что:
* Процесс преобразования кода в исполняемый файл выглядит примерно: Компиляция → объектные файлы → линковка → прошивка.
* Перед main() идёт инициализация памяти и периферии.
* static в функции сохраняет значение между вызовами.
* Основная задача разработчика — расписать требования и сделать расширяемый дизайн ПО.
