= Типы данных и преобразования в C++ для микроконтроллеров

== 1. Основные типы данных

[cols="2,2,6",options="header"]
|===
| Тип | Размер | Описание
| bool | 1 байт | Логическое значение true / false.
| char | 1 байт | Символ ASCII. Используется в строках.
| unsigned char | 1 байт | Беззнаковый символ (альтернатива `std::byte`).
| int | 4 байта | Стандартное целое число.
| unsigned int | 4 байта | Беззнаковое целое.
| float | 4 байта | Число с плавающей точкой.
| double | 8 байт | Число двойной точности.
|===

=== Модификаторы типов

- short int — 2 байта
- unsigned short int — 2 байта
- long int — 8 байт
- unsigned long int — 8 байт
- long double — 8 байт

== 3. Преобразования типов

=== Неявное преобразование

*Происходит автоматически*:

[source,cpp]
----
float f = 10; // int → float
----

=== Явное преобразование

*Требует явного указания*:

[source,cpp]
----
int x = 10;
float f = static_cast<float>(x);
----

== 4. Операторы преобразования типов

[cols="2,6",options="header"]
|===
| Оператор | Назначение
| static_cast | Безопасное преобразование совместимых типов
| reinterpret_cast | Переопределение типа указателя (опасно)
| dynamic_cast | Преобразование между типами с виртуальными функциями
| const_cast | Изменение константности объекта
|===

[NOTE]
====
reinterpret_cast использовать только в особых случаях — возможны ошибки!
====

== 5. Указатели

- Указатель хранит адрес переменной.
- Разность указателей допустима только в рамках одного массива или объекта.
- Вычитание указателей, указывающих на разные объекты, приводит к неопределённому поведению.

== 6. Битовые операции

[cols="2,6",options="header"]
|===
| Операция | Выражение
| Установить бит | x |= (1 << n);
| Сбросить бит | x &= ~(1 << n);
| Переключить бит | x ^= (1 << n);
|===

== 7. Унарные операторы

- +x — возвращает само значение без изменений.
- -x — меняет знак на противоположный.

== 8. Почему 17000000 + 1 == 17000000 для float?

Число 17000000 (`1.7e7`) в формате float представляется с точностью около 7-8 значащих цифр.  
При прибавлении 1 изменений в значении не происходит, потому что разница меньше разрешённой точности формата.

[plantuml, float_precision, svg]
----

start
:float x = 1.7e7;
:x = x + 1;
if (разница заметна?) then (нет)
:x остаётся равным 1.7e7;
else (да)
:значение меняется;
endif
stop
@enduml
----

== Краткий итог

- Размер типов зависит от архитектуры и компилятора.
- Явные преобразования — контроль за безопасностью кода.
- Осторожно с разностью указателей и использованием reinterpret_cast.
- Помнить о точности при работе с числами с плавающей точкой (`float`).
