== Цель

Цель отчета – узнать, что такое ОС, ОСРВ, FreeRTOS и т.д.

== Определение ОС:

*Операционная система* — это основа взаимодействия пользователя и аппаратного обеспечения.

=== Понятие операционной системы

*Операционная система (ОС)* — это комплекс программ, управляющих ресурсами компьютера и обеспечивающих выполнение пользовательских и системных приложений.

|===
| *Функции ОС* | *Описание*

| Процессорное управление | Распределение времени между процессами
| Управление памятью      | Выделение и освобождение ОЗУ
| Файловая система        | Создание, чтение и управление файлами
| Ввод/вывод              | Работа с устройствами (клавиатура, дисплей и др.)
| Интерфейс               | Командная строка или графическая оболочка
|===

== Операционные системы реального времени (ОСРВ)

ОСРВ ориентированы на задачи, где критична точность времени исполнения.

====
В системах реального времени задержки могут быть фатальными. Поэтому ОСРВ разрабатываются с прицелом на *детерминированность* и *минимальные задержки*.
====

=== Классификация ОСРВ

|===
| Вид | Описание

| *Жесткого реального времени (Hard RTOS)* 
| Тайминги должны соблюдаться строго. Нарушение — критично (например, управление двигателями в самолёте).

| *Мягкого реального времени (Soft RTOS)* 
| Допускаются небольшие временные отклонения. Пример — мультимедийные проигрыватели.
|===

== Многозадачность

Многозадачность — способность ОС выполнять несколько задач одновременно.

*Варианты реализации:*

* Кооперативная — задача передаёт управление добровольно;
* Вытесняющая — ОС сама решает, когда прервать выполнение.

== Планировщик

Планировщик — "дирижёр" системы, распределяющий ресурсы между задачами.

=== Критерии планирования:

* Приоритеты;
* Время ожидания;
* Политика ОС.

В системах реального времени используется *вытесняющий приоритетный* планировщик.

=== Приоритетный вытесняющий планировщик

В ОСРВ часто применяется приоритетный вытесняющий планировщик:

* Каждая задача имеет приоритет — выше приоритет — значит более высокий приоритет исполнения.
* Если появляется задача с более высоким приоритетом, текущая задача вытесняется.
* Планировщик переключается на задачу с самым высоким приоритетом из готовых.
* Для предотвращения голодания может использоваться повышение приоритета (aging).

=== Пример работы приоритетного вытесняющего планировщика

* Запускается задача с низким приоритетом.
* Появляется задача с более высоким приоритетом — текущая задача вытесняется.
* После завершения высокоприоритетной задачи продолжается выполнение низкоприоритетной.

== FreeRTOS

FreeRTOS — лёгкая операционная система реального времени с открытым исходным кодом.

|===
| Характеристика | Описание

| Компактность | Ядро занимает минимум памяти
| Портируемость | Поддержка множества микроконтроллеров (ARM, AVR и др.)
| API | Удобные функции управления задачами, очередями, мьютексами и др.
|===


=== Структура проекта

До текущего момента настройка касалась исключительно таймера *TIM2*. Однако следует учитывать, что каждый таймер обычно привязывается к своему отдельному прерыванию. При увеличении числа таймеров и прерываний (например, при n ≤ m, где n — количество прерываний, а m — доступное количество таймеров микроконтроллера) такой подход становится неэффективным.
Причина в том, что для обработки каждого таймера потребуется отдельное условие в коде, что приведёт к его усложнению, ухудшит читаемость и затруднит отладку системы.
В данном проекте код изначально интегрирован с операционной системой реального времени FreeRTOS через обёртку RtosWrapper. Она скрывает низкоуровневые детали взаимодействия с FreeRTOS, обеспечивая более чистую и понятную архитектуру приложения.
* Рабочая директория: cortex/RtosWrapper
* Задачи описываются в директории: cortex/RtosWrapper/MyTasks (если смотреть с папки проекта).

Обёртка RtosWrapper позволяет скрыть детали настройки FreeRTOS и сделать код более модульным и удобным для дальнейшего расширения.
== Межзадачное взаимодействие (МЗВ)

*МЗВ* — механизмы обмена информацией и синхронизации между задачами.

=== Типы взаимодействия:

* Очереди
* Семафоры
* Мьютексы
* События
* Потоки сообщений
* Уведомления задач

== Задачи

Задача (Task) — наименьшая исполняемая единица в ОСРВ.

В FreeRTOS:

* Создаётся через xTaskCreate()
* Имеет приоритет
* Выполняется с вытесняющей многозадачностью

=== Пример задачи

[source,cpp]
----
#include "MeasureTask.hpp"
#include "adc.h"
#include <iostream>

void MeasureTask::Execute()
{
  Adc<std::uint32_t> adc;
  adc.TurnOn();
  for (;;)
  {
    adc.StartConversion();
    adc.WaitEndOfConvertion();
    const auto adcCode = adc.GetData();
    
    const std::uint32_t tsCal1Address = 0x1FFF7A2CU;
    const std::uint16_t tsCal1 = *reinterpret_cast<std::uint16_t*>(tsCal1Address);
    
    const std::uint32_t tsCal2Address = 0x1FFF7A2EU;
    const std::uint16_t tsCal2 = *reinterpret_cast<std::uint16_t*>(tsCal2Address);
    
    const auto k = (110.0f - 30.0f) / static_cast<float>(tsCal2 - tsCal1);
    const auto b = 110.0f - k * static_cast<float>(tsCal2);
    
    const auto temp = k*adcCode + b;
    const auto filteredTemp = mFilter.FilterValue(temp);
      
    std::cout << "Temperature: " << temp << std::endl;
    std::cout << "Filtered temperature: " << filteredTemp << std::endl;
    
    Sleep(mSampletime);
  }
};
----

Эта задача предназначена для:
* Периодического измерения температуры с помощью встроенного датчика и АЦП.
* Корректного пересчёта кода АЦП в температуру с учётом заводских калибровок.
* Фильтрации результата для устранения шумов.
* Вывода результата в консоль (скорее всего, для отладки).

== Отключение планировщика и прерываний

Такие действия допустимы:

* Во время инициализации
* В критических секциях
* При необходимости предотвратить переключение задач

====
Отключения должны быть минимальными по времени, иначе возможны нарушения таймингов.
====

== Мьютексы

Мьютекс (mutex) — средство синхронизации, предназначенное для защиты ресурсов от одновременного доступа.

Преимущества:

* Только одна задача может владеть мьютексом
* Поддержка *наследования приоритета*
* Возможность создания *рекурсивных* мьютексов

== Дедлок (Deadlock)

====
Дедлок — состояние, при котором задачи навсегда ожидают друг друга, блокируя работу системы.
====

=== Причины дедлок:

* Циклическое ожидание
* Неправильный порядок захвата ресурсов

=== Методы предотвращения:

* Тайм-ауты при захвате ресурсов
* Единый порядок доступа к ресурсам
* Избежание вложенных блокировок
